Hello Hexo

很久以前就开了自己的blog，只是一直太懒，十几年来才写几篇blog。
最近发现Github Pages被人用来做博客，觉得很新鲜。
一般的blog都是CMS系统，只能按既定的格式进行文章的发布。Github pages是个静态页面的server，这样一来便可以完全掌握自己的博客了。

我以前研究过一些CMS的源码，一般都是动态页面，其中部分项目有提供了动态页面静态化方法，用于减轻服务器的压力。
Github仅提供静态网页Web服务，所以要么自己手写HTML，要么就找到HTML生成器。
旁边有小伙伴使用Hexo来进行Markdown->HTML的操作，似乎还比较主流。我也尝试一把。
于是便有了这篇博文。

关于Markdown语法

由于Hexo是使用Markdown文件来生成HTML的，我又懒得装Markdown编辑器（至少现在懒得装）。只能手写了。
我在公司弄了个doku wiki作为部门wiki之用，Doku wiki也用Markdown，但和Media Wiki的语法不同，它并非标准的Markdown语法。
造成了现在又得重新学习标准的Markdown。
